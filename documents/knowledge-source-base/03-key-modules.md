## Overview

BoomOpen Workflow Kit operates as a "content framework" â€” the majority of its modules are Markdown/YAML files interpreted by AI coding tools. Only two modules contain executable code: the CLI installer (`cli/`) and the web marketing site (`web/`). All other modules are declarative configuration consumed by AI assistants at prompt-interpretation time.

### Routing Logic
The Orchestrator (defined in `rules/CORE.md`) routes commands as follows:
1. User invokes `/command` or `/command:variant`
2. Orchestrator loads `commands/{command}.md` as the router
3. If variant specified, loads `commands/{command}/{variant}.md`
4. Natural language mapping: "implement/build/create" â†’ `/cook` or `/code`, "fix/bug" â†’ `/fix`, "plan" â†’ `/plan`
